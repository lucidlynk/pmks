7. ALUR BATCH PENGAJUAN
----------------------------------------------------------------
DRAFT -> SUBMITTED -> VERIFIED -> APPROVED
              |                       |
           REJECTED           REVISION_REQUESTED
              |                       |
         (operator)              (operator)
         reopen->DRAFT          accept->REVISED->SUBMITTED

Status Aksi:
- DRAFT              : Operator edit & submit
- SUBMITTED          : Verifikator verify/reject
- VERIFIED           : Admin approve/reject
- APPROVED           : Admin request_revision
- REJECTED           : Operator reopen -> DRAFT
- REVISION_REQUESTED : Operator accept -> REVISED
- REVISED            : Operator edit & submit ulang

Aturan Delete Batch (hanya Admin Dinsos):
- BOLEH hapus  : DRAFT, REJECTED, REVISION_REQUESTED, REVISED
- TIDAK BOLEH  : SUBMITTED, VERIFIED, APPROVED

8. VALIDASI PMKS
----------------------------------------------------------------
Aturan validasi DISIMPAN DI DATABASE (bukan hardcoded):
Tabel  : pmks_categories
Kolom  : min_age, max_age (null=tanpa batas), gender_restriction (L/P/null)
Class  : App\Rules\PmksAgeRule        (validasi usia dari DB)
Class  : App\Rules\DisabilityTypesRule (wajib isi untuk kategori disabilitas)

Kategori dengan validasi:
- PMKS-01           : 0-5 tahun
- PMKS-02 s/d 07    : 6-18 tahun
- PMKS-08           : 60 tahun ke atas
- PMKS-23           : Perempuan saja

9. HAL PENTING FILAMENT V4
----------------------------------------------------------------
- Section HARUS dari: Filament\Schemas\Components\Section
  (BUKAN Filament\Forms\Components\Section)
- $view adalah instance property (bukan static) di Page
- $navigationIcon bertipe: string|BackedEnum|null
- $navigationGroup bertipe: string|UnitEnum|null
- Form method di Page menerima: Schema $schema (bukan Form $form)
- Policy deleteAny() yang dicek Filament (bukan canDeleteAny() di Resource)
- databaseNotifications() diaktifkan di AdminPanelProvider
- Relation::morphMap didaftarkan di AppServiceProvider untuk PsksSubmission

----------------------------------------------------------------
10. STATUS PMKS/PSKS
----------------------------------------------------------------
Status PMKS dan PSKS TIDAK punya kolom sendiri.
Status derive dari batch induknya via accessor:

  public function getStatusAttribute(): string
  {
      return $this->batch?->status->value ?? 'draft';
  }
